# 需求说明（B 站博主视频维护与绝版库）

## 1. 目标与范围
- 目标：持续维护一批 B 站博主的视频，自动拉取新视频到本地并长期积累“绝版视频”。
- 范围：
  - 维护博主列表（UID 或名称）。
  - 定期检查新视频并下载保存到本地。
  - 对已收录视频定期检查是否下架，标记为“绝版”。
  - 超过一定时间未下架的视频标记为“稳定”。
  - 本地存储超过限制时触发清理，优先保留“绝版”。
- 非范围（当前版本）：
  - 多平台（抖音/快手/小红书）采集。
  - App 端（仅后端服务）。

## 2. 术语
- 博主（Creator）：B 站用户，可用 UID 或名称识别。
- 视频（Video）：博主发布的单个视频。
- 绝版（Out-of-print）：视频已被下架或不可访问。
- 稳定（Stable）：超过设定阈值仍未下架的视频。
- 采集（Fetch）：获取博主视频列表及视频元数据。

## 3. 功能性需求
### 3.1 博主管理
- 支持添加、更新、删除、启用/停用博主。
- 允许以 UID 或名称添加；名称需解析为 UID 并缓存。
- 支持导入/导出博主列表（CSV/JSON）。

### 3.2 新视频发现与拉取
- 周期性拉取博主最新视频列表。
- 识别新视频（未入库）并生成下载任务。
- 拉取视频时保存：标题、简介、发布时间、封面、分区、时长、播放/点赞等统计（如可获取）。
- 处理重复任务，保证幂等。

### 3.3 视频生命周期与状态
- 状态机（简化）：
  - `NEW` → `DOWNLOADING` → `DOWNLOADED`
  - `DOWNLOADED` → `OUT_OF_PRINT`（下架）
  - `DOWNLOADED` → `STABLE`（超过阈值仍未下架）
  - 任意状态 → `DELETED`（被清理）
- 每个视频建立独立检查计划（定时器）。

### 3.4 下架检测
- 定期检查视频可访问性，默认频率为每天 1 次。
- 若下架，标记 `OUT_OF_PRINT` 并记录下架时间。
- 若超过稳定阈值仍未下架，标记 `STABLE`，默认阈值为 30 天。

### 3.5 存储与清理
- 支持配置本地存储上限（如 2TB）。
- 超限时执行清理策略：
  - 绝版优先保留。
  - 优先删除稳定/低价值/重复视频。
  - 清理权重按顺序考虑：是否绝版、博主粉丝量、播放量、收藏量。
  - 可配置清理策略权重。

### 3.6 查询与管理
- 支持按博主、状态、时间区间检索视频。
- 提供任务状态与失败原因查询。
- 提供系统存储占用与清理历史。

## 4. 非功能性需求
- 可用性：单机可运行，支持随时恢复与重试。
- 幂等性：同一视频不会重复下载、重复入库。
- 扩展性：后续可接入多平台（插件化）。
- 性能：支持数千博主规模，下载与检查可并发配置。
- 可观测性：日志 + 指标（任务成功率、下载耗时、失败率）。
- 配置化：调度周期、下架阈值、存储上限均可配置。

## 5. 约束与假设
- 以合法合规方式访问公开内容。
- 默认单租户、单后端服务。
- 视频文件存储在本地文件系统。

## 6. 验收标准（示例）
- 添加博主后，系统在一个调度周期内能够发现并下载新视频。
- 下架视频在一个检查周期内被标记为 `OUT_OF_PRINT`。
- 超过阈值未下架的视频会自动标记 `STABLE`。
- 当存储超过限制时，系统在一次清理流程内释放到安全阈值以下。

## 7. 待确认问题
- 下架检测的接口方式与成功率要求。
- “稳定”阈值与检查频率的默认配置。
- 清理策略优先级与权重（是否考虑播放量/收藏等指标）。
