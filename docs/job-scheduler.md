# 调度与任务设计

## 1. 任务类型
- fetch_creator_videos：拉取博主视频列表。
- download_video：下载视频文件。
- check_video_availability：检查视频可访问性。
- cleanup_storage：触发存储清理。

## 2. 默认调度策略（建议）
- 拉取新视频：每 30-60 分钟一次（可配置）。
- 下架检查：
  - 默认频率：每天 1 次。
  - 超过“稳定阈值”（默认 30 天）仍可访问 → 标记 `STABLE`。
- 清理任务：
  - 每日凌晨或超限触发。

## 3. 幂等与重试
- 任务去重键：
  - fetch: (creator_id, schedule_window)
  - download: (video_id)
  - check: (video_id, scheduled_at)
- 失败重试：指数退避，最大重试次数可配置。
- 任务超时：下载任务设定超时并可恢复。

## 4. 并发控制
- 下载并发、检查并发单独配置。
- 同一博主下载可限制并发，避免频繁请求。

## 5. 限速与礼貌策略
- 支持全局 QPS 限制与随机抖动。
- 避免高峰集中请求。

## 6. 任务状态与可观测性
- 记录任务状态变化与失败原因。
- 输出关键指标：
  - 新视频发现数/天
  - 下载成功率
  - 下架检测命中率
  - 存储清理频次与释放容量
