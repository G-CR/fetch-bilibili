# 前端驾驶舱设计规格

## 背景

当前仓库已有一个 `web/` 静态控制台原型，但结构和视觉语言都偏临时实现，难以继续扩展。用户要求删除旧静态页面，改用前端框架重做，并采用 `Raycast` 风格的单页工作台。

## 目标

构建一个与 Go 后端解耦的前端应用，作为独立工作台使用，满足以下要求：

- 使用框架开发，而不是继续维护手写静态页
- 视觉方向为 `Raycast` 风格，深色、锐利、克制，辅以 B 站粉蓝强调
- 首页为 `总览驾驶舱优先`
- 布局为 `左侧边栏 + 主工作区`
- 页面组织为 `单页驾驶舱`，通过锚点导航浏览不同区块
- 第一版以界面完整和交互结构为主，允许部分模块继续使用前端占位数据

## 非目标

- 第一版不补充新的后端业务接口
- 第一版不做 SSR 或复杂状态管理
- 第一版不追求完整数据闭环
- 第一版不沿用旧 `web/` 的 DOM 结构和样式体系

## 信息架构

### 左侧边栏

- 品牌区：产品名称、连接模式、状态提示
- 导航锚点：`总览`、`博主`、`任务`、`存储`、`风控`、`设置`
- 快捷动作：拉取、检查、清理

### 主工作区

- 顶部命令栏：标题、说明、主要动作按钮
- 总览区：核心指标卡片与系统状态概览
- 运行态区：任务队列、执行状态、最近事件
- 博主管理区：快速添加表单与列表
- 存储区：容量、清理优先级、风险提示
- 风控区：限速、退避、Cookie 建议
- 设置区：API 模式、本地模式、地址配置

## 视觉规范

### 整体气质

- 不做营销型大 Hero
- 不堆叠大面积发光和漂浮玻璃卡
- 使用冷黑背景、精细边线、局部亮面与少量高亮色
- 保持高信息密度但避免拥挤

### 色彩

- 基础色：深黑灰、煤灰、冷灰蓝
- 强调色：B 站粉、B 站蓝
- 用法：强调色仅用于 CTA、选中状态、关键数值、图表点

### 组件风格

- 指标卡片采用紧凑面板式，而非大阴影卡片
- 列表更接近工作台行视图
- 按钮使用锐利的深色底与高对比描边
- 状态标签统一采用细边框、小字号、大写或半大写风格

### 动效

- 首屏指标区轻微进入动效
- 侧栏锚点切换有平滑高亮
- 列表和日志 hover 反馈克制

## 数据策略

第一版采用混合数据策略：

- 能使用现有 API 的模块继续联动
- 暂无接口的数据继续由前端状态模拟
- 保留清晰的 API 层，后续便于替换成真实接口

## 技术方案

- 新建独立前端目录 `frontend/`
- 使用 `React + Vite`
- 使用原生 CSS，不引入 UI 组件库
- API 封装独立于页面组件
- 不将前端代码嵌入 Go 服务内部

## 风险与边界

- 当前后端接口较少，总览区大量数据仍需前端模拟
- 容器化部署后续需增加独立前端服务或静态资源容器
- 若后续改为真实联动，需要补充 `/stats`、`/jobs list` 等接口
