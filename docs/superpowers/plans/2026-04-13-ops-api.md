# 运维接口补全实现计划

> **面向 AI 代理的工作者：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法来跟踪进度。

**目标：** 补齐基础运维接口，让前端和运维人员可以对单个博主、单个视频和存储清理进行直接操作，而不是只能依赖全局调度。

**架构：** 继续沿用现有 `CreatorService`、`JobService`、`DashboardService` 三层接口，不引入新模块。`PATCH /creators/{id}` 走博主服务部分更新；`GET /videos/{id}` 走 dashboard 查询；`POST /videos/{id}/download` / `POST /videos/{id}/check` 通过任务服务入队，其中单视频检查复用 `check` 任务并携带 `video_id` payload，worker 按 payload 只检查目标视频；`POST /storage/cleanup` 直接入队 cleanup。

**技术栈：** Go、net/http、MySQL、sqlmock、单元测试
