# 浏览器级 E2E 联调测试计划

> **面向 AI 代理的工作者：** 本轮先补测试基础设施，再补脚本和断言，最后执行验证并同步 TODO。优先保证仓库内可重复执行，同时保留真实环境联调入口。

**目标：** 为独立前端驾驶舱补齐浏览器级 E2E，覆盖“打开页面、同步数据、添加博主、触发任务”四条主链路。

**设计决策：**
- 使用 `Playwright`。
- 提供两种模式：
  - `mock`：启动本地 mock API 服务，验证前端与现有接口契约的浏览器级联动。
  - `live`：直接访问用户本机正在运行的前后端地址，做真实联调。
- 默认执行 `mock`，确保仓库内可重复跑通。
- `live` 模式通过环境变量切换，不强依赖本仓库启动后端。
